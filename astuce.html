Étape 1 Comment créé un dépot Gitub
Il faut donc se connecter sur Github .
Une fois connecté sur Github aller sur le + en haut puis aller sur New repository .
Choissir le nom de votre projet à validé . Puis faire la commande suivante :
git clone git@github.com:le/nom/du/dépot.git

Étape 2 Créé une clef SSH

Pour créé un clef SSH réaliser la commande suivante
ssh-keygen -t ed25519 -C "mon@mail.com"
Suivre le proces et coppier le contenue de la clef
Aller sur Github > settings >SSH and GPG keys > puis généré une clef

Étape 3 Créé une clef PGP

Pourquoi créé un clef PGP : La PGP pemetra de signé vos commit prouvant que c'est bien vous qui la réalisé Pour un créé la clef PGP , il nous faut aller sur notre Github Photo de profil > Setings(Paramètre) > Dans la colone de gauche "SSH and GPG keys" > New GPG key (Nouvelle clef GPG)
Il faut après généré la clef , mais pour ça il faut GnuPG .
Pour Arch Linux veulez tappez la mcommande pour installer GnuPG
udo pacman -S gnupg
Une fois instalé nous allons généré une clef GPG avec la commande :
gpg --full-generate-key
Nous allons par la suite un certain nombre de questionne voila ce que nous allons répondre :
Pour la clef désiré nous allons prendre soit la 10 pour juste avoir un signature soit la 9 pour avoir la signature plus d'autre sécurité
Pour le type de courbe elliptique désiré nous allons prend la 1
Pour le temps pendant lequel cette clef devrait être valable c'est à vous de choisir . Moi j'ai mis 2y ( 2 ans) Si tout est correcte metre O
Il faut après metre votre nom d'utilisateur Github
Puis votre adresse mail et si vous le désirez un commentaire
Une fois terminer que tout est bon ne metter pas Q mais bien O sinon allez doir tout recommencer !
Il faut donc un mot de passe pour sécurisé le tout . Je vous consils de le généré par votre gestionnaire de mot de passe mais ne trainait pas trop si , vous devrait tout recommencer .
ENFIN , nous avons notre celf GPG dans /.gnupg/openpgp-revocs.d
